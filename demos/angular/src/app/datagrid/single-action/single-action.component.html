Dev notes for single action grids

Single action is a simplified variation of the multi-select pattern. The main difference is that single selection does not need a form to manage which rows are selected for batch action. Single selection does need to implement a popover that can handle the row actions available for the row. Then, add a cds-action button to the row. When the rows are generated the application can pass the click event and the row model to a function that can handle associateing the button clicked with the popover element. Store a reference to the selected for of your model type to create a general way to access te row information that needs action on.
