Highlights

  • Design Tokens and Layouts

    Clarity has added layout utilities and design tokens for layout, spacing, color, and typography to @clr/core. Clarity Core's layout utilities can be used in plain HTML, although they are still considered experimental at this time and may feature breaking changes in the near future. (#4407).

  • Angular component for the Timeline

    An angular-based timeline component was added to supplement the existing HTML/CSS timeline component introduced in 2.2.0. (#4350)

  • Enhanced clrIfError in Clarity Forms

    The clrIfError structural directive has been enhanced to provide error objects for more robust error handling in Clarity Forms. (#4346)

Bug Fixes

  • Accessibility: type error on focus initialization (#4291)
  • Fix for tabs' keyboard focus navigation in Angular 9 (#4323)
  • Fix for id="null" appearing in clr-buttons (#4352)
  • Fixed calculation of Clarity updates in Angular schematics (#4341)
  • Fix for the stepper's first panel opening unexpectedly (#4353)
  • Fix for Jest and Nx not transforming the core common strings service to ES5 (#4404)
  • Fix for form inputs' errors announcing error messages to screen readers when there was no error (#4408)
