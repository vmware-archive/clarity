# Design Tokens

Design Tokens are variables that represent the foundation of the design system.
Changes to a design token will propagate visual changes throughout the entire system.
Tokens are organized into three categories:

- **Global**: global primitives (color, space, typography)
- **Alias**: values describing intent (status, success, error)
- **Component**: component specific values

Global tokens provide design primitives such as [Spacing](./?path=/docs/foundation-spacing-getting-started--page),
[Layout](./?path=/docs/experimental-layout-get-started--page), [Typography](./?path=/docs/experimental-typography-getting-started--page),
and [Color](./?path=/docs/experimental-color-getting-started--page). Alias tokens
are variables with described intent, concept or abstraction. Alias values should
always be prefered over global and component tokens. Component tokens are values
that are highly specific to a component instance. These values should be rarely
changed in the system.

```scss
// raw value
hsl(93, 80, 23)

// global token
--cds-global-color-green-800 

// alias token
--cds-alias-color-status-success

// component token
my-element {
  --background: some-color;
}
```

When using Tokens with Clarity on the Web we represent the design tokens using
CSS Custom Properties. CSS Custom Properties are variables that allow us to
dynamically change CSS values in the browser.

## Usage

Tokens can be adjusted by setting the values on the `:root` CSS selector.

```css
:root {
  --cds-global-typography-body-color: hsl(...);
}
```
